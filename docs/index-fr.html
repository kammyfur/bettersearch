BetterSearch:cloud, l'API officiel de BetterSearch

bettersearch://

BetterSearch, et son API BetterSearch:cloud vous permettent une meilleure exploitation de toutes les fonctionnalités et de la technologie que vous propose le moteur de recherche BetterSearch. En plus, il est gratuit !

Vous pouvez recréer la partie frontend du moteur de recherche selon vos envies, pour par exemple l'intégrer à vos propres services. Pensez quand même à nous faire un peu de publicité, ça nous ferait extrêmement plaisir !

Voici les adresses que vous devez appeler pour accéder aux différentes fonctions de BetterSearch:cloud. L'API ne retourne pas de JSON, ni de code exploitable par un logiciel facilement. Il retourne un message commençant par ok= si tout s'est bien passé (avec l'URL du résultat après le =), ou autre chose si une erreur s'est produite.

Utilisation

/api/quiet

Utilisez cette adresse pour retourner directement une redirection JavaScript vers la page de résultats.

Arguments GET

q : Requête à effectuer. Vous pouvez remplacer les espaces par des + si besoin
lang : Identifiant de la langue à utiliser pour la recherche, parmi les langues disponibles sur le client.
region : Identifiant du pays à utiliser pour la recherche, parmi les pays disponibles sur le client.

/api/search

Utilisez cette adresse pour retourner le résultat d'une Recherche Classique vers l'adresse cible.

Arguments GET

q : Requête à effectuer. Vous pouvez remplacer les espaces par des + si besoin
lang : Identifiant de la langue à utiliser pour la recherche, parmi les langues disponibles sur le client.
region : Identifiant du pays à utiliser pour la recherche, parmi les pays disponibles sur le client.

/api/search-corp

Utilisez cette adresse pour retourner le résultat d'une Recherche Entreprise vers l'adresse cible.

Arguments GET

q : Requête à effectuer. Vous pouvez remplacer les espaces par des + si besoin
lang : Identifiant de la langue à utiliser pour la recherche, parmi les langues disponibles sur le client.
region : Identifiant du pays à utiliser pour la recherche, parmi les pays disponibles sur le client.

/api/suggestions

Utilisez cette adresse pour rechercher les suggestions correspondantes à un terme de recherche. Il s'agit de la seule fonction de l'API à utiliser du JSON car le résultat doit être géré par le client.

Arguments GET

q : Termes à rechercher dans la base de données des suggestions
mode : Base de données de suggestions (search ou corps) à utiliser pour la recherche
limit : Nombre maximal de résultat demandés (plus il y en a, plus le retour sera long à venir), 5 par défaut

Conditions d'utilisation

Les conditions d'utilisation de BetterSearch:cloud qui s'appliquent sont les mêmes que celles de BetterSearch, excepté que vous devez créditer (au moins dans le code de votre client) le fait que vous exploitez l'API du logiciel BetterSearch.

Si vous êtes une personne gentille et que vous avez envie de soutenir le projet, vous pouvez créditer BetterSearch sur la page d'accueil de votre site, sur la barre de recherche, ou encore sur la page des mentions légales. Il n'y a actuellement aucune obligation pour vous de créditer l'utilisation de BetterSearch dans l'interface, mais obligation de le créditer au moins dans le client.
